# Product Spec

## 概要

本アプリは、ローカルPC上の文書ファイルを対象にした **全文検索中心の Web アプリ** である。  
本文抽出できない画像などは、補助的にファイル名検索で扱う。

利用形態は以下を想定する。

- 通常はローカルPC上で利用
- 必要に応じて Tailscale VPN 経由で私的アクセス
- インターネット公開はしない

## 目的

- 文書内容から目的のファイルを高速に見つける
- Google風のシンプルなUIで検索できるようにする
- ローカル用途に適した軽量構成にする
- macOS / Windows 両対応にする

## 主要要件

- 本文検索中心で、本文抽出しない一部ファイルはファイル名でも検索できる
- 日本語など空白で区切られない連続文字列は、部分語でも検索できる
- 検索時に対象フォルダのフルパスを指定できる
- 既存インデックスを対象に、全 DB 検索もできる
- 検索時に階層数を指定できる
- 検索前に対象フォルダのインデックスを必要時のみ更新できる
- 差分更新できる
- 同一対象を短時間で連続検索する場合、一定時間は更新を省略できる
- 対象拡張子を UI から選択できる
- 作成日 / 更新日 / アクセス数で並び替えできる
- 除外キーワード、同義語、対象拡張子設定を保存できる
- パスは OS 差異を吸収して正規化する
- ローカル専用または Tailscale 経由の私的アクセスを前提とする

## 技術方針

- Backend: Python + FastAPI
- Frontend: React + Vite
- Search: SQLite FTS5
- 日本語検索は FTS5 本文検索に加えて補助 bi-gram インデックスを併用する

## 非対象

- OCR

- 一般公開向け認証・認可
- Elasticsearch 導入

## 補足

現行コードでは、事前に検索対象フォルダを登録する方式は採用していない。  
検索時に `フルパス + 階層数 + 対象拡張子 + 除外キーワード` を指定し、その組み合わせを内部キャッシュとして管理する。

現行コードでは、日本語など空白区切りにならない本文の部分一致を拾うため、  
本文セグメントに加えて補助 bi-gram セグメントもインデックスする。  
旧インデックスに補助セグメントが存在しない場合は、検索時に再インデックスして補完する。

現行コードでは、通常配布時はビルド済み `frontend/dist/` を FastAPI から同一オリジン配信する。  
バックエンドの既定 bind は `127.0.0.1:8079`、`start_dev.sh` では `0.0.0.0:8079` を既定にできる。  
Vite 開発サーバを使う場合は `5173` 番ポートを使い、`allowedHosts` に `mac-mini` を含めている。

現行コードでは、除外キーワード・同義語・拡張子設定は DB ではなく `data/` 配下のテキストファイルへ保存する。
検索対象フォルダ、非表示にしたインデックス済みフォルダ、Obsidian sidebar-explorer のパス設定も `data/` 配下のテキストファイルへ保存する。
端末依存でよい既定検索フォルダだけはフロントエンドの `localStorage` に保持する。

現行コードでは、デスクトップランチャーを `launcher/` 配下で管理する。macOS は PyObjC / Cocoa `NSPanel`、その他 OS は Flet を使い、バックエンドから子プロセスとして起動・停止・ログ確認できる。システムトレイ常駐は未実装。
