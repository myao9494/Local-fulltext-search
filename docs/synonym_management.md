# 同義語管理仕様

## 概要
本アプリケーションでは、検索クエリに含まれる単語に対して同義語展開を行い、関連するキーワードでもヒットするようにしています。
このドキュメントでは、同義語の設定形式および取得APIについて記述します。

## 同義語の設定形式
同義語は設定ファイル（デフォルト: `synonym_groups.txt`）に記述されます。

- **形式**: 1行を1つの同義語グループとして扱います。
- **区切り文字**: カンマ（`,`）または全角カンマ（`，`）を使用します。
- **重複排除**: 同一グループ内および全体で重複は自動的に排除されます。
- **大文字小文字**: ASCII文字については、検索時は大文字小文字を区別せずに同一語として扱われます。

### 例
```text
PC,パソコン,パーソナルコンピュータ
iPhone,アイフォン,アイフォーン
Google,グーグル
```

## 同義語取得API
設定された同義語を構造化されたデータとして取得するためのAPIエンドポイントを提供します。

### エンドポイント
`GET /api/index/synonyms`

### レスポンス形式
```json
{
  "groups": [
    ["PC", "パソコン", "パーソナルコンピュータ"],
    ["iPhone", "アイフォン", "アイフォーン"],
    ["Google", "グーグル"]
  ]
}
```

## 実装の詳細
同義語のパースロジックは `IndexService._parse_synonym_groups` に実装されており、APIはこのロジックを再利用してデータを返します。
これにより、検索エンジンが使用しているのと全く同じパース結果をUI側でも確認することが可能です。
