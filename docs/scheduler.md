# スケジューラー仕様

## 概要
- ハンバーガーメニューの `スケジューラー` から設定画面を開く。
- 対象パスは複数登録できる。
- 開始日時は 1 つだけ指定する。
- 開始時刻になると、別プロセスで各フォルダを順次インデックスする。
- Windows では、手動設定とは別に 10:00 / 12:15 / 16:00 の固定時刻で、登録済みの有効な「検索対象フォルダ」を自動で全階層再インデックスする。

## 実行ルール
- スケジュール開始時に既存ログを初期化する。
- ログには開始、各フォルダの完了、失敗を記録する。
- 実行中もアプリ本体は使い続けられるよう、監視はメインプロセス、インデックス本体は子プロセスで動かす。
- フォルダ間の並列プロセス化は SQLite 競合とネットワーク負荷悪化を避けるため、現時点では行わず順次実行とする。
- Windows 固定スケジュールはバックグラウンドの子プロセスで実行し、同じ日付・同じ時刻枠では二重起動しない。
- Windows で前回実行中のままアプリが終了していた場合は、起動時に実行中履歴を解除し、再開ではなく次のスケジュール実行で一から再インデックスする。
- macOS では Windows 固定スケジュールの判定・起動を行わず、既存の手動スケジューラー挙動だけを維持する。

## 現在の前提
- スケジューラー実行時のインデックス深さは全再帰相当の固定値で処理する。
- 除外キーワードと対象拡張子は既存のアプリ設定をそのまま利用する。
