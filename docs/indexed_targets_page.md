# インデックス済みフォルダ管理ページ

## 目的
- 実際にインデックス済みファイルを含むすべてのフォルダを一覧で確認する
- キーワードで対象を絞り込み、複数選択してインデックスを削除する

## 画面仕様
- 画面上部のタブから `インデックス済みフォルダ` ページへ切り替える
- 一覧には以下を表示する
  - フルパス
  - インデックス済みファイル数
  - 最終インデックス日時
- 一覧は `targets` の管理レコードではなく、`files` テーブルから祖先フォルダを展開して構成する
- `キーワードで絞り込み` はフルパスを対象に部分一致で絞り込む
- `すべて選択` は現在の絞り込み結果に含まれる行だけを対象にする
- `選択したフォルダのインデックスを削除` は確認ダイアログ後に実行する

## API 仕様
- `GET /api/index/targets`
  - `files` から展開したインデックス済みフォルダ一覧を返す
- `DELETE /api/index/targets`
  - `folder_paths` に含まれるフォルダ配下のインデックスを削除する
  - 対象フォルダ配下の `files` / `file_segments` / `failed_files` も削除する
  - 重なる `targets` は `last_indexed_at = NULL` に戻し、次回検索で再インデックスさせる
