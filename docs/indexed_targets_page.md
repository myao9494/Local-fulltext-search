# インデックス済みフォルダ管理ページ

## 目的
- 実際にインデックス済みファイルを含むすべてのフォルダを一覧で確認する
- キーワードで対象を絞り込み、複数選択してインデックスを削除する
- 検索対象フォルダ（インデックス作成の許可対象）を有効/無効で管理する

## 画面仕様
- 画面上部のタブから `インデックス済みフォルダ` ページへ切り替える
- ページ内タブを `インデックス済みフォルダ` / `検索対象フォルダ` で切り替える
- 一覧には以下を表示する
  - フルパス
  - インデックス済みファイル数
  - 最終インデックス日時
- 一覧は `targets` の管理レコードではなく、`files` テーブルから祖先フォルダを展開して構成する
- `キーワードで絞り込み` はフルパスを対象に部分一致で絞り込む
- `すべて選択` は現在の絞り込み結果に含まれる行だけを対象にする
- `選択したフォルダのインデックスを削除` は確認ダイアログ後に実行する
- `検索対象フォルダ` タブでは、各行のチェックボックスで有効/無効を切り替えられる
- `検索対象フォルダ` タブでは、行単位で `削除`、`インデックス削除`、`インデックス再取得` を実行できる
- `インデックス取得` は選択中または有効な検索対象フォルダをまとめてインデックスする
- 起動URLの `full_path` が検索対象フォルダに含まれない場合、検索画面に `検索対象に追加` ボタンを表示する
- 既に親フォルダが検索対象なら、子フォルダは追加候補として扱わない

## API 仕様
- `GET /api/index/targets`
  - `files` から展開したインデックス済みフォルダ一覧を返す
- `DELETE /api/index/targets`
  - `folder_paths` に含まれるフォルダ配下のインデックスを削除する
  - 対象フォルダ配下の `files` / `file_segments` / `failed_files` も削除する
  - 重なる `targets` は `last_indexed_at = NULL` と `indexed_file_count = 0` に戻し、次回検索で再インデックスさせる
- `GET /api/index/search-targets`
  - 検索対象フォルダ一覧（`is_enabled` を含む）を返す
- `PUT /api/index/search-targets`
  - `folder_path` 単位で検索対象の有効/無効を更新する
- `POST /api/index/search-targets`
  - 検索対象フォルダへ新規追加し、有効状態にする
- `POST /api/index/search-targets/reindex`
  - 指定した検索対象フォルダ群を順次再インデックスする
- `DELETE /api/index/search-targets`
  - 指定した検索対象フォルダを一覧から削除する
